Servidores virtuales (EC2)

Almacenamiento de datos (S3)

Bases de datos (RDS, DynamoDB)

Antes de empezar debemos saber las categorías de servicios en la nube:


IaaS (Infrastructure as a Service): Infraestructura básica (servidores, redes, almacenamiento). Ej: AWS EC2, Google Compute Engine.

PaaS (Platform as a Service): Plataforma lista para desarrollar y desplegar apps. Ej: Heroku, Google App Engine.

FaaS (Function as a Service): Funciones de código que se ejecutan bajo demanda (serverless). Ej: AWS Lambda, Azure Functions.

SaaS (Software as a Service): Aplicaciones listas para usar vía internet. Ej: Gmail, Microsoft 365, Dropbox.

IAM

En AWS, el servicio IAM (Identity and Access Management) es una plataforma centralizada que te permite controlar de forma segura quién tiene acceso a tus recursos y qué puede hacer con ellos

No se recomienda tocar servicios con la cuenta root, la cuenta root es la primera cuenta que nos hemos creado, en el dashboard es la que sale arriba a la derecha, siempre se recomienda hacerlo con una cuenta IAM
